Linux & Bash

What is Linux?

When we think about Linux, we usually really mean the Linux kernel. The kernel is the core of an operating system, interacting directly with the computer's hardware components. Linux is more of a family of operating systems than one single operating system, but every Linux OS has the Linux kernel at its core. We call all the operating systems that use the Linux kernel, "distributions" or "flavors."

Tux, the official brand character of the Linux kernel.


Linux Distributions

Linux distributions come in all shapes and sizes. Linux is ultimately an incredibly powerful and customisable operating system, and we can see this reflected in the variety of distributions available.

Popular distributions include:

  • Debian- One of few independent Linux distrubtions made from scratch; uses the APT package manager.
  • Ubuntu - A distribution based on Debian and composed mostly of free and open-source software; slightly more user friendly that Debian.
  • Kali - A Debian-derived distribution designed for digital forensics and penetration testing.
  • Arch - A distrubtion known for minimal distribution-specific changes--and therefore minimal breakage with updates--and a custom package manager, Pacman.
  • CentOS - A free and open-source community distribution that provides compatibility with its upstream source, Red Hat Enterprise Linux.
  • Parrot - A distribution based on Debian with a focus on security, privacy, and development.

The Shell

Now that we have a general idea of what Linux is, let’s start learning how to use it! To use any OS, you need to give it commands to tell it what to do. You are probably familiar with clicking buttons and dragging things around on the screen to tell your OS what operations to perform. When you do this, you are using what is called the graphical user interface (GUI, pronounced gooey).

There is another way to give your OS commands, however: the command-line interface (CLI). Instead of clicking buttons with the GUI, to use the CLI you type commands to tell the OS what actions to perform. The CLI is often quicker to navigate and more powerful than the Linux GUI. The Linux CLI is powered by the shell--a program that takes the commands you type in the CLI and sends them to the OS to perform. There are several different shells that accept slightly different commands, but for this course and virtual machine, we will focus on one called bash (Bourne Again shell).

Most Linux distributions will default to the bash shell. Depending on which distribution you’re working with, the shell prompt might change slightly. But, for the most part, it should be something along these lines:

joe@uca:/home/joe $

The prompt includes the $, indicating you're logged in as a normal user.
